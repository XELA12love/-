# 系统运行与验收说明

## 1. 环境要求

- Node.js 18+（前端 `admin`/`client`）
- JDK 8（后端 `back-stage`）
- MySQL 5.7+/8.0
- 后端端口默认：`9999`

## 2. 启动顺序

1. 启动数据库（确保库和表已初始化）
2. 启动后端 `back-stage`
3. 启动后台前端 `admin`
4. 启动用户前端 `client`

## 3. 开发启动命令

### 后端

```bash
# 在 back-stage 目录
# 方式一：IDE 直接运行 SpringBoot 主类
# 方式二：如果本机有 Maven
mvn spring-boot:run
```

### 后台前端

```bash
# 在 admin 目录
npm install
npm run dev
```

### 用户前端

```bash
# 在 client 目录
npm install
npm run dev
```

## 4. 生产构建命令

```bash
# admin
npm run build

# client
npm run build
```

## 5. 验收清单（建议按顺序）

### 后台验收

1. 访问后台登录页，验证码可点击刷新
2. 登录后左侧菜单可切换，顶部主题切换可用
3. 进入用户/商品/订单/公告/分类/轮播页面，表格加载正常
4. 各模块新增/编辑弹窗样式正常，提交成功
5. 推荐看板数据可加载，TOP 列表可刷新

### 用户端验收

1. 首页：搜索、分类跳转、推荐区、热销区展示正常
2. 商品列表：筛选与分页正常
3. 商品详情：轮播图、下单弹窗、收藏、评论正常
4. 推荐页：策略切换、刷新、点击埋点请求正常
5. 公告列表与公告详情页跳转正常
6. 登录/注册/个人中心功能可用

## 6. 常见问题排查

### 1) 提示“禁止访问/未授权”

- 先确认后端是否启动成功
- 确认前端请求前缀与后端上下文一致（当前是 `/springbootGWyfHA`）
- 若登录后仍 401，清理浏览器本地存储后重新登录

### 2) 图片不显示

- 检查 `back-stage/src/main/resources/application.yml` 的 `file.upload-dir` 路径是否存在、可读写
- 检查后端 `/upload/**` 映射是否可访问

### 3) 本机无法编译后端

- 如果提示 `mvn` 不存在，说明本机未安装 Maven
- 可改用 IDE 直接运行 SpringBoot 主类，或安装 Maven 后再编译
